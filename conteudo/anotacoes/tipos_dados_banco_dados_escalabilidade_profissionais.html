Tipos de Dados

Quais os tipos de dados são utilizados em ciência de dados:

Dados estruturados

  • Possui esquema fixo;
  • Formato bem definido ;
  • Conhecimento prévio da estrutura de dados;
  • Simplicidade para relacionar informações;
  • Dificuldade para alterar o modelo.

Ex:tabelas de um banco de dados, banco de dados semântico.

Dados não estruturados

  • Sem tipo predefinido;
  • Não possui estrutura regular;
  • Pouco ou nenhum controle sobre a forma;
  • Manipulação mais simplificada;
  • Facilidade de alteração modelo ;
  • Modelo flexível;

Ex: textos, documentos, chave valor



Bancos de Dados

Quais tipos de bancos de dados podem ser usados:

Banco de dados Relacionais (SQL)

Prós

Banco de dados relacionais são bons para dados estruturados e transacionais.

Possuem baixo tempo de resposta e são capazes de realizar consultas complexas.

Contras

Pode ser dificil de escalar;

Esquema fixo pode dificultar a organização e evolução dos dados.

Ex.: Oracle, MySQL, PostgreSQL, SQL Server

Banco de Dados NoSQL

Prós

Bom para dados não relacionais;

Arquitetura sem esquema permite mudancas frequentes;

Escalabilidade mais fácil

Roda bem em ambientes distribuidos

Contras

Ferramentas ainda em fase de amadurecimento;

Incompatibilidade com SQL;

Ex.:MongoDB, elastic Search, HBase, CouchBase, Redis, Cassandra, CosmosDB.



Banco de dados NewSQL

Tentativa de unificar as melhores ideias do mundo NoSQL e mundo relacional

Profissionais

Quais os profissionais que atuam com dados:

Data Scientist / Cientista de dados

O data scientist tem como principal função validar hipóteses e extrair Insights a partir dos dados.

Atribuições:

  • Data mining;
  • Sistemas de recomendação;
  • Machine Learning;
  • Inteligência Artificial;
  • Modelagem estatística;
  • Métodos de testes.

Skills:

  • Matemática;
  • Estatística;
  • Visualização de dados;
  • Machine learning;
  • Programação.

Data Engineer / Engenheiro de dados

O data engineer garante o fluxo continuo entre as fontes de dados e os sistemas de armazenamento e processamento. São responsáveis por definir a arquitetura dos dados.

Atribuições:

  • Transformações de dados;
  • Processamento paralelo;
  • Integração de sistemas heterogêneos;
  • Construir aplicações escaláveis;
  • Analise de performance.

Skills:

  • Programação avançada;
  • Design de banco de dados;
  • Otimização.
